# IDEF0 DSL — обзор проекта

## Что это

Текстовый DSL для моделирования бизнес-процессов в нотации IDEF0, оптимизированный под хранение в git и редактирование в обычном текстовом редакторе. Модель читается непосредственно из исходников без рендеринга в графику.

## Архитектура

Три слоя:

1. **DSL** — текстовый формат `.idef0` файлов. Спецификация в `spec/01-dsl.md`. Раскладка файлов по папкам — `spec/03-layout.md`. Структура подпапок внутри проекта валидатором игнорируется и удобна для пользователя (ей следует плагин и ИИ-агент). Граница проекта определяется расположением маркерного файла `A0.*.idef0`.
2. **Валидатор** — парсит файлы, собирает модель, валидирует её, выдаёт ошибки и предупреждения. Спецификация в `spec/04-validator.md`. Рендер в SVG и другие визуальные форматы не делается; ASCII-схема живёт в будущем форматтере.
3. **VS Code расширение** — UX-обёртка вокруг валидатора и форматтера. Отображает `.idef0` и сгенерированную ASCII-схему в едином UI через Notebook API, даёт подсветку синтаксиса с семантической раскраской ролей стрелок и форматирование по команде. Решения по MVP первой итерации — `sagas/idefy-mvp/mvp-spec.md`.

**Текущий статус:** Discovery первой итерации MVP закрыта (`sagas/idefy-mvp/mvp-spec.md`). Спецификации языка, форматтера, раскладки и валидатора (`spec/`) готовы. Реализация расширения ещё не начата.

## Спека

- `spec/01-dsl.md` — формат содержимого `.idef0` файлов: синтаксис, семантика, правила именования.
- `spec/02-formatting.md` — правила форматирования `.idef0` (строгое ТЗ для форматтера).
- `spec/03-layout.md` — физическая раскладка проектов в файловой системе: структура папок, корневая папка проекта, имена.
- `spec/04-validator.md` — что делает валидатор: правила, ошибки, предупреждения.
- `spec/05-classical-gaps.md` — фичи классической IDEF0, осознанно пропущенные.
