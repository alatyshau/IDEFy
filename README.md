# IDEF0 DSL — обзор проекта

## Что это

Текстовый DSL для моделирования бизнес-процессов в нотации IDEF0, оптимизированный под хранение в git и редактирование в обычном текстовом редакторе. Модель читается непосредственно из исходников без рендеринга в графику.

## Архитектура

Три слоя:

1. **DSL** — текстовый формат `.idef0` файлов. Спецификация в `spec/01-dsl.md`. И также есть спецификация расположения файлов по папкам в `spec/03-layout.md`. Раскладка внутри проекта не влияет на компилятор, но удобна для пользователя (ей следует плагин и ИИ-агент). Граница проекта определяется расположением маркерного файла `A0.*.idef0`.
2. **Компилятор** — парсит файлы, валидирует модель, выдаёт ошибки и предупреждения. Спецификация в `spec/04-compiler.md`. Рендер в SVG/другие визуальные форматы в первой версии **не делается**.
3. **Плагин VS Code** — управляет физической раскладкой файлов и папок, служит форматтером. Спецификация в `spec/05-plugin-spec.md`.

**Текущий статус:** Есть только спека, и по ней ИИ может уже генерить idef0-файлы и красиво раскладывать по папкам. Реализация компилятора и VS Code пока не начата. И пока не решили нужно ли вообще реализовывать, т.к. пока спеки достаточно.

## Спека

- `00-README.md` — этот файл.
- `01-dsl.md` — формат содержимого `.idef0` файлов: синтаксис, семантика, правила именования.
- `02-formatting.md` — рекомендации по форматированию исходного кода `.idef0`.
- `03-layout.md` — физическая раскладка проектов в файловой системе: структура папок, корневая папка проекта, имена.
- `04-compiler.md` — что делает компилятор: правила валидации, ошибки, предупреждения.
- `05-plugin-spec.md` — VS Code плагин: поведение, форматирование, раскладка.
- `06-classical-gaps.md` — фичи классической IDEF0, которые мы пока пропустили.
