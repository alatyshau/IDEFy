# Layout — раскладка проекта

## Корневая структура

Компилятор сканирует исключительно каталог `src/idef0/`. Всё, что лежит
вне этого пути, для компилятора не существует.

Каждый проект (см. `01-dsl.md`) — связное поддерево `.idef0` файлов под
`src/idef0/`. Корневая папка проекта определяется наличием в ней
`A0.*.idef0`. Имя проекта вычисляется по пути от `src/idef0/` до
корневой папки проекта в Java-нотации:

- `src/idef0/coffee/brewing/A0.idef0` → проект `coffee.brewing`
- `src/idef0/banking/lending/as_is/A0.idef0` → проект `banking.lending.as_is`

Имена папок на пути от `src/idef0/` до корневой папки проекта обязаны
быть валидными идентификаторами Java-пакета (lowercase ASCII, цифры,
подчёркивание; без точек). Полные правила — в `01-dsl.md`, раздел «Имя
проекта».

Между `src/idef0/` и корневой папкой проекта обязан быть минимум один
уровень вложенности — `src/idef0/A0.idef0` запрещён.

## Структура внутри проекта

Каждый проект — это одна IDEF0-модель. В корневой папке проекта лежат:

- **`A-0.*.idef0`** — контекстный файл (обязательный). Содержит плоский
  список туннелей и опциональную ссылку на A0.
- **`A0.*.idef0`** — корневой процесс (обязательный, маркер проекта).
- **`Aₓ.*.idef0`** для каждого процесса, у которого есть декомпозиция.
  Атомарные процессы (без декомпозиции) собственного файла **не имеют** —
  они существуют только как строки-дети в файле родителя.

## Свобода организации внутри проекта

**Компилятор полностью игнорирует структуру подпапок внутри проекта.**
Он рекурсивно сканирует все `.idef0` файлы поддерева и работает только
с именами файлов и их содержимым. Подпапки — это инструмент для людей.

Это значит, что все следующие раскладки **эквивалентны** для компилятора:

### Плоская раскладка

```
src/idef0/coffee/brewing/
    A-0.idef0
    A0.idef0
    A1.idef0
    A11.idef0
    A2.idef0
    A3.idef0
```

### Вложенная по иерархии декомпозиции

```
src/idef0/coffee/brewing/
    A-0.idef0
    A0.idef0
    A0/
        A1.idef0
        A1/
            A11.idef0
        A2.idef0
        A3.idef0
```

### Группировка по бизнес-фазам

```
src/idef0/coffee/brewing/
    A-0.idef0
    A0.idef0
    intake/
        A1.idef0
        A11.idef0
    processing/
        A2.idef0
        A3.idef0
```

Все три собираются в одну и ту же модель. Команда выбирает раскладку по
удобству и согласует её через VS Code плагин (см. `04-plugin-spec.md`).

## Косметические имена файлов и папок

Внутри корневой папки проекта (на любой глубине) в имени любого файла
и любой папки после ID процесса может быть произвольное человекочитаемое
описание после первой точки. Компилятор его игнорирует.

```
A0.Приём_заявок.idef0       # эквивалентно A0.idef0
A11.Проверить.idef0          # эквивалентно A11.idef0
A1.Принять/                  # папка эквивалентна A1/
```

Косметика — чисто для удобства навигации в IDE и при чтении `git log`.

**Важно:** косметика разрешена только **внутри** проекта. На пути от
`src/idef0/` до корневой папки проекта точки в именах папок запрещены
(см. `01-dsl.md`, раздел «Имя проекта»).

## Несколько проектов в одной области

Если в одной бизнес-области нужны несколько моделей (например, AS-IS
и TO-BE), они организуются как **отдельные проекты** в соседних папках:

```
src/idef0/banking/lending/as_is/
    A-0.idef0
    A0.idef0
    ...

src/idef0/banking/lending/to_be/
    A-0.idef0
    A0.idef0
    ...
```

Полные имена процессов: `banking.lending.as_is::A0` и
`banking.lending.to_be::A0`. Каждый проект самостоятелен, id-пространства
изолированы.

Использовать `B0`, `C0` и подобные префиксы для разных моделей запрещено
(всегда только `A0` как корень). «A» — это convention IDEF0 для Activity
diagram, не индикатор «первой модели».

## Расширение под другие методологии

Если в будущем понадобятся другие методологии (IDEF1X, IDEF3, что-то иное),
они получают отдельный корневой каталог `src/idef1x/`, `src/idef3/` и т.д.
со своей структурой. В одной папке IDEF0-файлы не смешиваются с файлами
других методологий.

## Правила компилятора по раскладке

- Файлы вне `src/idef0/` компилятором не рассматриваются.
- Имя файла `<ID>[.<косметика>].idef0` обязано следовать формату.
- ID в имени файла должен совпадать с id в шапке процесса/контекста внутри.
- Глобальная уникальность ID в пределах проекта: два файла с одинаковым
  ID где угодно в проекте — ошибка коллизии.
- Папка с именем `<ID>[.<косметика>]/` внутри проекта — допустима, но её
  имя ни на что не влияет (компилятор не смотрит).
- `.idef0` файл, не принадлежащий ни одному проекту (сирота), — ошибка.
