# Плагин VS Code — спецификация

## Назначение

Плагин для VS Code предоставляет команде удобный UX для работы с
`.idef0` моделями: подсветка синтаксиса, автоматическая раскладка файлов,
интеграция с компилятором.

В первой версии — минимально необходимый набор функций.

## Целевая аудитория

Команда из 3 человек. Плагин делается «для себя». Не предполагается
публикация в Marketplace в первой версии.

## Функции первой версии

### Раскладка файлов

Плагин управляет физической раскладкой `.idef0` файлов в файловой системе
по заранее заданному правилу — **одна захардкоженная стратегия**. Стратегия
вынесена в отдельный модуль/файл плагина, чтобы при необходимости заменить
её было дело нескольких минут.

Стратегия определяет:

- Как располагаются файлы по подпапкам внутри проекта.
- Как именуются файлы (с какой косметикой, в каком формате).
- Как именуются папки.

Конкретная стратегия первой версии — на усмотрение реализации. Возможные
варианты (плоская, иерархическая, по фазам) описаны в `02-layout.md`.
Главное — стратегия должна быть **одна и стабильная**.

### Форматирование

Плагин действует как форматтер `.idef0` файлов. При сохранении или по команде
он:

- Приводит файл к каноническому виду (отступы, пустые строки, выравнивание).
- При необходимости перекладывает файл в нужную папку согласно текущей
  стратегии раскладки.
- Переименовывает файл, если ID процесса в шапке изменился.

### Интеграция с компилятором

Плагин запускает компилятор при сохранении или по запросу и отображает
ошибки/предупреждения в стандартной панели Problems.

## Принципы дизайна

### Раскладка — это форматирование

Раскладка по папкам полностью эквивалентна форматированию: компилятор не
смотрит на структуру папок, поэтому плагин может произвольно перекладывать
файлы без риска сломать модель. Это аналог `prettier` или `gofmt` для кода.

### Расширяемость отложена

Множественные пресеты раскладки, кастомные правила, конфигурация через
файл в репозитории — всё это **out of scope** для первой версии. YAGNI.

Если возникнет потребность во второй стратегии — она добавляется простым
расширением модуля стратегии. Параметризация через настройки откладывается
до момента, когда возникнет реальная необходимость.

### Безопасность не приоритет

Поскольку плагин «для себя», вопросы безопасности (произвольное выполнение
кода из конфигурации, RCE-векторы и т.п.) не рассматриваются. Стратегия —
часть кода плагина, не конфигурация.

## Что плагин не делает

- Не рендерит модель в SVG или другие визуальные форматы.
- Не предоставляет графический редактор диаграмм.
- Не валидирует модель самостоятельно (вся валидация — в компиляторе).
- Не публикуется в Marketplace.
- Не предлагает настройки раскладки пользователю.
