# Форматирование и стиль IDEF0 DSL

Этот документ описывает стиль написания и форматирования исходного кода `.idef0`, который не является строгим для компилятора, но используется для поддержания читаемости и может автоматически применяться плагином-форматтером (например, в VS Code).

## Короткие объявления (в одну строку)

Если объявления потребляемых и производимых стрелок короткие, их рекомендуется писать в одну физическую строку с выравниванием имён, двоеточий `:`, переходов `->` и терминаторов `;` в аккуратные колонки:

```idef0
A1 "Помол зерна"     : I1, C1, M1, M2                 -> X11 "Смолотый кофе";
A2 "Подготовка воды" : I2, C1, M3                     -> X21 "Вода нужной температуры";
A3 "Экстракция"      : I[X11], I[X21], C1, C2, M1, M4 -> X31[O1], X32[O2];
```

## Переносы длинных объявлений

Благодаря тому, что каждое логическое объявление обязательно завершается точкой с запятой `;`, компилятор не зависит от физических строк. Вы можете осуществлять переносы там, где вам удобно (как в Java или C).

Однако для сохранения читабельности кода рекомендуются переносы:
- после двоеточия `:` и после стрелки `->`
- после запятых в списках потребляемых или производимых стрелок

**Важно:** Перенос внутри ID стрелки, внутри ID активности или внутри строкового литерала запрещён.

```idef0
A1 "Помол зерна" :
    I1,
    C1,
    M1,
    M2 ->
    X11 "Смолотый кофе",
    X12[T1];

A3 "Экстракция" : I[X11], I[X21], C1, C2, M1, M4 ->
    X31[O1],
    X32[O2];
```
