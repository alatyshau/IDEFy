# Форматирование и стиль IDEF0 DSL

Этот документ описывает стиль написания и форматирования исходного кода `.idef0`, который не является строгим для компилятора, но используется для поддержания читаемости и применяется плагином-форматтером (например, в VS Code) как строгое ТЗ.

## Граничные стрелки

1. Объявления граничных стрелок (интерфейс активности) всегда должны размещаться строго по одной стрелке на каждую строку.
2. Между блоком граничных стрелок и списком функциональных блоков декомпозиции должна быть ровно **одна пустая строка**.

```idef0
activity A0 "Работа с Креатором" {
    I1 "set process project";
    O1 "show named process project";
    C1 "PSL";

    A1 "Проектирование САПР" ...
```

## Функциональные блоки

Блоки внутри декомпозиции всегда **сортируются по их ID** (от `A1` до `A9`, `AA`..`AZ`, затем внуки по той же логике). Пользователь управляет порядком блоков, задавая им нумерацию. Если пользователь изменит номер (например, `A2` на `A5`), авто-форматтер обязат переместить строку в соответствии с сортировкой.

Для форматирования функциональных блоков предусмотрено **три режима канонического расположения**:

### Режим 1: В одну строку

Используется для коротких объявлений. Примыкающие друг к другу блоки в Режиме 1 группируются без пустых строк.

**Правила выравнивания и группировки (Табличный вид):**
1. Двоеточия `:` выравниваются в единую вертикальную колонку с помощью пробелов/табуляции для всех блоков в Режиме 1 в файле.
2. Потребляемые стрелки слева от `->` группируются строго по ролям: `I`, затем `C`, затем `M`. Форматтер создает визуальные колонки под эти группы. Если группа (например, `C`) отсутствует в данном блоке, её место остаётся пустым, чтобы выровнять соседние группы.
3. Стрелки перехода `->` выравниваются в единую вертикальную колонку по самой широкой комбинации входов в файле.

```idef0
    A1 "Поиск"         : I1,     C2, C3, C[T2], M1, M3 -> X11 "Кандидаты";
    A2 "Интервью"      : I[X11], C3,            M2, M4 -> X21 "Оценки";
    A3 "Оффер"         : I[X21], C4, C[T1],     M1, M2 -> X31 "Оффер", X32[O2];
    A4 "Оформление"    : I[X31], C1, C2,        M1, M2 -> X41[O1];
```

### Режим 2: На три строки

Используется, когда объявление не помещается на экран, но списки стрелок до и после `->` достаточно короткие. Между блоком в Режиме 2 (или 3) и соседними блоками ставится **одна пустая строка**.
- На первой строке остаётся только ID и имя блока.
- На вторую строку переносится двоеточие `:` и все потребляемые стрелки. Добавляется один уровень отступа (табуляция) спереди. Стрелки разбиваются на группы `I`, `C`, `M`, но без жестких табличных колонок.
- На третью строку переносится `->` и все производимые стрелки. Используется тот же один уровень отступа.

```idef0
    A13 "Задание входов и выходов процесса" 
        : I[X112], I[X113], I[X114], C1 
        -> X131 "Спецификация входов и выходов процесса";
```

### Режим 3: Многострочный с переносом стрелок

Аналогично Режиму 2, но всё, что идёт после `->`, дополнительно переносится на последующие строки (по одной производимой стрелке на строку) с еще одним дополнительным уровнем табуляции.

```idef0
    A11 "Разбор проекта процесса" 
        : I1, C1
        ->
            X111 "set <process name>", 
            X112 "set <input>", 
            X113 "set <output>", 
            X114 "kind";
```

## Сортировка стрелок внутри групп

Внутри каждой роли (группы `I`, `C`, `O`, `M`) стрелки **обязательно сортируются**.
Правило сортировки: цифры и буквы идут перед символом `[`. 
Это гарантирует, что собственные (короткие) стрелки уровня `I1`, `I2`, `C4` выводятся первыми, а затем идут стрелки соседей `I[X...]` и туннели `I[T...]`.

Пример правильной сортировки:
`I1, I2, I[T1], I[X11], I[X22]`


## Правила выбора режимов (Триггеры авто-форматтера)

По умолчанию для всех функциональных блоков используется **Режим 1**. Переход к более сложным режимам происходит по следующим правилам:

1. **Триггер для Режима 2:**
   Если вычисленная длина строки блока в Режиме 1 (включая все выравнивающие табуляции/пробелы) превышает **120 символов**, блок переключается в **Режим 2**.
   *Примечание: Исключение этого блока из Режима 1 пересчитывает ширину колонок для оставшихся блоков в Режиме 1 (что делает их компактнее).*

2. **Триггер для Режима 3:**
   Если блок уже перешёл в Режим 2 (из-за длины) или если среди производимых стрелок (после `->`) есть **более одной стрелки с собственным описанием** (формата `X.. "Описание"`), то блок переключается в **Режим 3**. 
   *Примечание: Если порождается всего одна стрелка с описанием или несколько стрелок-наследников вида `X[O..]`/`X[T..]`, Режим 3 не триггерится (достаточно Режима 1 или 2).*


3. **Подрежим Б (Длинный список потребляемых стрелок):**
   Если блок находится в Режиме 2 или 3, и длина строки с потребляемыми стрелками (которая начинается с `: `) превышает **120 символов**, включается дополнительный перенос для левой части.
   В этом случае:
   - Двоеточие `:` остаётся на второй строке (с одним табом).
   - Каждая группа потребляемых стрелок (`I`, `C`, `M`) переносится на свою отдельную строку с дополнительным уровнем табуляции (два таба).
   - Если группа (строка) всё равно превышает 120 символов, дальнейший перенос внутри неё **не делается** (игнорируем, YAGNI).

   Пример:
   ```idef0
    A15 "Очень сложная активность с кучей входов"
        : 
            I1, I2, I[X111], I[X112], I[X113], I[X114],
            C1, C2, C[X121], C[X131],
            M1, M2, M3
        -> X151[O1];
   ```



**Важно:** Перенос внутри ID стрелки, внутри ID активности или внутри строкового литерала запрещён в любом из режимов.
